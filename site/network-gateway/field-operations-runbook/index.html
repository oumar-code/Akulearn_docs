    Network Gateway Field Operations Runbook¶

    Overview¶

    Operational guide for field agents, support teams, and administrators managing eSIM and physical SIM provisioning, activation, and troubleshooting.

    Audience: - Field agents performing SIM registration and activation - Support team resolving user issues - Network operations monitoring system health - Training coordinators onboarding new staff

    Part 1: Field Agent Operations¶

    1.1 Physical SIM Registration¶

    Prerequisites¶

    • Field agent app installed and logged in
    • SIM card inventory (pre-loaded with blank profiles)
    • ICCID barcode scanner (or manual entry)

    Step-by-Step Process¶

    Step 1: Verify SIM Inventory

    1. Open Field Agent App
2. Navigate to "Inventory" tab
3. Confirm SIM stock count matches physical inventory
4. If discrepancy, initiate stock reconciliation

    Step 2: Register SIM

    1. Tap "Register New SIM"
2. Scan ICCID barcode on SIM card
   - Format: 89 (country) + network code + unique ID (19-20 digits total)
   - Example: 89234123456789012345
3. If barcode scan fails:
   - Tap "Enter Manually"
   - Type ICCID carefully (verify twice)
4. System validates ICCID:
   - ✅ Green checkmark: ICCID valid and unused
   - ❌ Red X: ICCID invalid or already registered
5. Assign MSISDN (phone number):
   - System suggests next available number
   - Or enter specific number if user requests
6. Select initial plan:
   - Free Tier (default): Zero-rated Akulearn content
   - Starter Data: 500 MB/month + zero-rated
   - Premium: 2 GB/month + zero-rated
7. Tap "Activate SIM"

    Step 3: Hand to User

    1. Remove SIM from card (punch out gently)
2. Hand SIM to user
3. Instruct user to:
   - Power off phone
   - Insert SIM in SIM tray (correct orientation)
   - Power on phone
4. Wait 1-2 minutes for network registration

    Step 4: Verify Activation

    1. App shows "Activating..." status
2. After 1-2 minutes, status changes to:
   - ✅ "Activated" (success)
   - ⚠️ "Pending" (HLR delay, wait 5 more minutes)
   - ❌ "Failed" (see troubleshooting below)
3. If activated, instruct user to dial *123#
4. USSD menu should appear within 5 seconds
5. User selects "1 - Check Balance" to verify

    Common Issues & Fixes | Issue | Cause | Resolution | |-------|-------|------------| | ICCID already registered | Duplicate entry or recycled SIM | Check if SIM previously issued; if yes, call support to reassign | | Invalid ICCID format | Typo or damaged barcode | Re-enter manually; verify with printed ICCID on SIM | | Activation timeout (>5 min) | HLR/HSS overload | Wait 10 minutes; if still pending, escalate to support | | No USSD menu | Network registration delay | Wait 5 more minutes; verify signal strength (≥2 bars) |

    1.2 eSIM Profile Provisioning¶

    Prerequisites¶

    • Field agent app with eSIM module enabled
    • User has eSIM-capable device (iPhone XS+, Samsung S20+, Google Pixel 3+)
    • Stable internet connection for profile download

    Step-by-Step Process¶

    Step 1: Verify Device Compatibility

    1. Ask user for device model
2. Check compatibility list in app ("Device Checker" tab)
3. If device not listed or older than 2018, recommend physical SIM
4. If compatible, proceed to profile generation

    Step 2: Generate eSIM Profile

    1. Tap "Provision eSIM"
2. Select plan (same as physical SIM options)
3. App calls backend API: POST /esim/profiles
4. Profile generated in 5-10 seconds
5. QR code appears on screen

    Step 3: Display QR Code

    1. Show QR code to user
2. Instruct user to:
   iOS:
   - Settings > Cellular > Add Cellular Plan
   - Scan QR code displayed on agent's device
   - Tap "Add Cellular Plan"
   - Wait for download (10-30 seconds)
   - Tap "Continue" to activate

   Android (Samsung, Google):
   - Settings > Connections > SIM Manager
   - Add Mobile Plan
   - Scan QR code
   - Download profile
   - Set as default data SIM if prompted

    Step 4: Verify Activation

    1. Agent app shows "Downloading..." status
2. After 30-60 seconds, status changes to "Activated"
3. User sees new SIM label in device (e.g., "Akulearn eSIM")
4. Test network: Open web browser, visit akulearn.com
5. If loads successfully, activation complete

    Alternative: Manual Entry (No QR Scan)

    If user prefers or QR scan fails:
1. Agent taps "Show Manual Entry Details"
2. Display activation code screen showing:
   - SM-DP+ Address: smdp.akulearn.com
   - Activation Code: LPA:1$smdp.akulearn.com$ABCD1234-5678-90EF
3. User navigates to eSIM settings
4. Selects "Enter Details Manually"
5. Copies SM-DP+ address and activation code
6. Confirms and downloads profile

    Common Issues & Fixes | Issue | Cause | Resolution | |-------|-------|------------| | QR scan fails | Poor lighting or damaged screen | Use manual entry method | | "Invalid QR Code" error | Expired profile (>24 hrs old) | Generate new profile; old one expired | | Download stuck at 50% | Weak internet connection | Switch to WiFi; retry download | | "Unable to Activate" | Profile already downloaded once | Cannot reuse; generate new profile | | Device shows "No Service" | Profile downloaded but not activated | User must set as active SIM in settings |

    1.3 Device Swap Operations¶

    Scenario: User Switching Phones¶

    Physical SIM Swap:

    1. User removes SIM from old phone
2. Insert SIM into new phone
3. Field agent updates backend:
   - Open Field Agent App
   - Navigate to "SIM Management"
   - Search by ICCID or phone number
   - Tap "Update Device"
   - Enter new IMEI (scan from phone settings or packaging)
   - Tap "Save"
4. Backend calls POST /psim/iccids/{iccid}/swap
5. HLR updated with new IMEI
6. User powers on new phone
7. Network registration completes in 1-2 minutes

    eSIM Profile Swap:

    1. User's new phone must be eSIM-capable
2. Agent initiates swap:
   - Search for existing profile by phone number
   - Tap "Transfer to New Device"
   - Generate new QR code for new device
3. User scans QR on new device (follows eSIM activation steps)
4. Old device profile automatically deactivated within 90 seconds
5. Backend calls POST /esim/profiles/{id}/swap with new IMEI
6. swap.completed event fired to webhooks

    Important: Only one device can be active per ICCID/profile at a time. Swapping deactivates the old device.

    Part 2: Support Team Playbook¶

    2.1 Common User Issues¶

    Issue: "My phone says No Service"¶

    Diagnostics:

    Step 1: Verify SIM Status
- Ask for phone number or ICCID
- Search in support portal
- Check status:
  ✅ ACTIVE → Proceed to Step 2
  ⚠️ PENDING → HLR provisioning delay; wait 10 minutes
  ❌ SUSPENDED → User marked as lost/stolen; verify identity and unsuspend
  ❌ INACTIVE → SIM never activated; escalate to field agent

    Step 2: Network Troubleshooting

    Actions:
1. "Can you see signal bars on your phone?"
   - No signal: Phone in airplane mode or SIM not inserted properly
   - Signal present but no service: Continue to Step 3

2. "Go to Settings > Network > Carrier"
   - iOS: Should show "Akulearn" or partner MNO name
   - Android: Should show network name in status bar

3. Try manual network selection:
   - Settings > Network > Carrier Selection > Manual
   - Select Akulearn or partner MNO from list
   - If not in list, user outside coverage area

4. Restart phone:
   - Power off completely (hold power button 10 seconds)
   - Wait 30 seconds
   - Power on and wait 2 minutes for registration

    Step 3: Backend Verification

    Support Portal Actions:
1. Check recent events for ICCID:
   - Look for "attach.success" event in last 24 hours
   - If present, network registration working recently
   - If absent, SIM may not be inserted or device faulty

2. Check policy status:
   - Verify policy pushed successfully
   - If policy.pushed event missing, manually trigger:
     POST /policies/push with ICCID

3. Check HLR/HSS status:
   - Query HLR for ICCID
   - Verify MSISDN mapped correctly
   - If missing, escalate to network operations

    Resolution: - 70% of cases: Restart phone resolves issue - 20% of cases: Manual network selection fixes registration - 10% of cases: Backend policy push or HLR re-sync required

    Issue: "I can't download my eSIM profile"¶

    Diagnostics:

    Step 1: Check Device Compatibility
- "What phone model do you have?"
- Verify in compatibility matrix (see Section 3.4)
- If device not supported, offer physical SIM alternative

Step 2: Check QR Code Validity
- QR codes expire after 24 hours
- "When did you receive the QR code?"
- If >24 hours old, generate new profile:
  - Call POST /esim/profiles with same MSISDN
  - Mark old profile as EXPIRED
  - Send new QR code to user via SMS or email

    Step 3: Download Troubleshooting

    Common Causes:
1. Weak internet connection
   - "Are you connected to WiFi or mobile data?"
   - Recommend WiFi for initial download

2. Profile already downloaded once
   - eSIM profiles are one-time use
   - Cannot redownload same profile
   - Must generate new profile (backend handles deduplication)

3. Device storage full
   - eSIM profiles require 1-2 MB storage
   - "Check if device has available storage"
   - Free up space and retry

4. Carrier restrictions (rare)
   - Some devices locked to specific carriers
   - "Is your phone carrier-unlocked?"
   - If locked, contact original carrier to unlock

    Resolution: - 60% of cases: WiFi connection fixes download - 30% of cases: New profile generation required (expired QR) - 10% of cases: Device compatibility or lock issue

    Issue: "My data isn't working but calls are fine"¶

    Diagnostics:

    Step 1: Verify APN Settings
- APN (Access Point Name) configures data connection
- Incorrect APN = No data

iOS:
- Settings > Cellular > Cellular Data Network
- APN: akulearn.internet
- Username: (leave blank)
- Password: (leave blank)

Android:
- Settings > Network & Internet > Mobile Network > Access Point Names
- Tap "+" to add new APN
- Name: Akulearn
- APN: akulearn.internet
- APN type: default,supl
- Save and select as active APN

    Step 2: Data Roaming (If Applicable)

    If user on partner MNO network:
- Data roaming must be enabled
- Settings > Cellular/Mobile Data > Data Roaming: ON
- Explain: "You're using our partner network, so roaming must be on"

    Step 3: Data Usage Check

    Support Portal:
1. Check user's data balance: GET /data-usage/{iccid}
2. If balance = 0 and plan is not zero-rated:
   - User exhausted data bundle
   - Offer bundle purchase via USSD (*123*3#)
   - Or upgrade plan in support portal
3. If balance > 0 but data not working:
   - Check PGW logs for failed sessions
   - Verify policy enforcement not blocking incorrectly

    Resolution: - 80% of cases: APN configuration fixes data - 15% of cases: Data roaming setting resolves issue - 5% of cases: Backend policy or PGW issue

    Issue: "I lost my SIM card"¶

    Immediate Actions:

    Step 1: Suspend SIM (Security)
- Verify user identity:
  - Ask for phone number and name on account
  - Match with customer record in CRM
- If identity confirmed:
  - Mark SIM as LOST in support portal
  - PATCH /psim/iccids/{iccid} with status: LOST
- Backend suspends network access within 10 seconds
- Prevents unauthorized usage and billing

    Step 2: Replacement Options

    Option A: Found Original SIM
- User finds lost SIM later
- Reactivate: PATCH /psim/iccids/{iccid} with status: ACTIVE
- Network access restored in 20 seconds

Option B: Issue Replacement SIM
- Physical SIM: Issue new SIM with same MSISDN
  - Register new ICCID linked to same customer account
  - Old ICCID permanently marked as LOST
  - User inserts new SIM and activates
- eSIM: Generate new profile
  - Old profile automatically revoked
  - User downloads new profile via QR code

    Important: Only one SIM/profile can be active per MSISDN at a time.

    Issue Type First Contact Resolution Escalation Level 1 Escalation Level 2 Escalation Level 3
    No network service Support agent (restart, APN config) Network operations (HLR check) Engineering (PCEF/PGW logs) Vendor (MNO support)

    Data not working Support agent (APN, roaming settings) Network operations (policy check) Engineering (PGW session logs) -
    Billing discrepancy Support agent (usage query) Billing team (reconciliation) Finance (adjustment approval) -

    3.2 Error Codes Reference¶

    Code Message Cause Resolution
    4001 Invalid ICCID format ICCID not 19-20 digits or wrong format Verify ICCID printed on SIM; re-enter carefully
    4002 ICCID already registered Duplicate registration attempt Check if SIM previously issued; reassign if needed
    4003 MSISDN not available Phone number already assigned Choose different MSISDN or release old assignment
    4004 Invalid plan selection Plan code not recognized Use one of: free-tier, starter-data, premium
    4005 Profile download failed SM-DP+ unreachable or timeout Retry profile generation; check user's internet
    5001 HLR provisioning timeout HLR/HSS not responding Wait 10 minutes; if persists, escalate to network ops
    5002 Policy push failed PCEF not reachable Manually retry: POST /policies/push
    5003 Network attach failed Device not registering to network Check device signal strength; try manual network selection

    Part 5: Incident Response Procedures¶

    5.1 High-Priority Incidents¶

    Incident: HLR/HSS Outage (P1 - Critical)¶

    Symptoms: - All SIM activations failing - Existing SIMs losing network registration - attach.failed events flooding logs

    Immediate Actions: 1. Network ops paged via PagerDuty 2. Check HLR/HSS health dashboard 3. If down, initiate failover to backup HLR (if available) 4. If no backup, contact MNO partner for emergency HLR access

    Communication: - Post incident notice on status page (status.akulearn.com) - Email all field agents: "SIM activations temporarily unavailable" - Estimate time to resolution (ETA)

    Resolution: - Primary HLR restored: Resume activations - Backlog of pending activations processed (queue-based) - Post-mortem within 48 hours

    Incident: Policy Enforcement Failure (P1 - Critical)¶

    Symptoms: - Zero-rated content being billed - Users charged for Akulearn data usage - Billing discrepancies reported

    Immediate Actions: 1. Suspend all policy pushes to prevent further incorrect billing 2. Identify affected users: Query billing system for zero-rated charges 3. Manually correct billing records (refund incorrect charges) 4. Investigate root cause: PCEF configuration, policy DB corruption, etc.

    Communication: - Urgent notification to billing team - Email affected users with apology and refund confirmation - Internal incident report to executive team

    Resolution: - Fix policy engine configuration - Re-push policies to all affected SIMs - Audit billing for 48 hours to ensure no recurrence - Implement additional policy validation checks

    5.2 Medium-Priority Incidents¶

    Incident: Elevated Activation Failure Rate (P2 - High)¶

    Symptoms: - Activation success rate drops from 98% to <90% - No complete outage, but increased failures

    Investigation: 1. Check for patterns: Specific device types? Specific regions? 2. Review recent deployments: Backend changes? Network config changes? 3. Check HLR/HSS load: High CPU or memory usage? 4. Check SM-DP+ latency: Profile download timeouts?

    Mitigation: - If HLR overloaded: Scale horizontally (add HLR instance) - If SM-DP+ slow: Switch to backup SM-DP+ endpoint - If backend API issue: Roll back recent deployment

    Resolution: - Restore success rate to >95% - Root cause analysis and corrective action plan - Deploy fixes to prevent recurrence
