    Aku Network Gateway — Physical SIM¶

    Overview¶

    Physical SIM (pSIM) support complements esim.md so feature phones and lower-spec smartphones can access Aku voice/data services. The network gateway must deliver near-parity with eSIM policies while keeping activation simple for users without app stores or QR-capable cameras.

    Goals¶

    • Serve non-smartphone and low-end Android users with a removable SIM option.
    • Maintain consistent network policies (local breakout first, MNO fallback) across pSIM and eSIM.
    • Support migrations both ways: pSIM ➜ eSIM (for capable devices) and eSIM ➜ pSIM (for device loss/downtime).

    Device support¶

    • Feature phones (2G/3G/4G) that can register on partner MNO bands used by the gateway.
    • Low-end smartphones without eSIM chips.
    • Compatibility matrix maintained in IG-Hub; activation rejects devices that fail band/lock checks.

    Provisioning & activation¶

    • Inventory: pre-printed SIMs with ICCID + PIN/PUK; serialized in IG-Hub.
    • KYC: validate subscriber identity prior to SIM handover (align with local NCC/MNO rules).
    • Activation: agent enters ICCID + subscriber into IG-Hub; gateway assigns policy and pushes to HLR/HSS equivalent.
    • OTA updates: limited; prefer USSD/SMS menus for balance, bundle purchase, and diagnostics.
    • SIM swap: support ICCID swap workflow to move a subscription from lost pSIM to new card or to eSIM.
    sequenceDiagram
    participant Agent
    participant IG as IG-Hub
    participant NG as Network Gateway
    participant Dev as Device (pSIM)
    Agent->>IG: Register ICCID + subscriber (KYC)
    IG->>NG: Push ICCID policy + breakout prefs
    Agent->>Dev: Hand SIM (ICCID, PIN/PUK)
    Dev->>NG: Attach + IMSI/ICCID auth
    NG->>Dev: Policy + USSD/SMS menu endpoints
    NG->>IG: Activation status + KPIs

    Activation error handling¶

    ICCID registration failures:

    flowchart TD
    START[Register ICCID] --> KYC_CHECK{KYC valid?}
    KYC_CHECK -->|Pass| DUPLICATE{ICCID exists?}
    KYC_CHECK -->|Fail| ERR1[400: KYC failed]
    DUPLICATE -->|No| STOCK{In stock inventory?}
    DUPLICATE -->|Yes| ERR2[409: ICCID already registered]
    STOCK -->|Yes| REGISTER[Register to HLR/HSS]
    STOCK -->|No| ERR3[404: ICCID not in inventory]
    REGISTER -->|Success| ACTIVATE[Mark active]
    REGISTER -->|Fail| RETRY{Retry count?}
    RETRY -->|< 3| REGISTER
    RETRY -->|>= 3| ERR4[503: Gateway unavailable]

    Device attach failures: - PIN locked: Verify PUK provided; unlock SIM; document in field agent runbook - IMSI mismatch: Check ICCID entered correctly; verify SIM not swapped before activation - Network not found: Device may not support MNO bands; check compatibility matrix; swap to supported device - Auth timeout: Gateway HLR/HSS unreachable; retry with exponential backoff; escalate if persistent

    SIM lifecycle state machine¶

    stateDiagram-v2
    [*] --> Inventory: SIM printed
    Inventory --> Registered: Agent registers ICCID
    Registered --> Active: Device first attach
    Registered --> Expired: 14 days no attach
    Active --> Suspended: Payment/admin suspend
    Active --> Lost: User reports lost
    Suspended --> Active: Resume
    Lost --> Swapping: Swap to new ICCID
    Swapping --> Active: New SIM attached
    Swapping --> Inventory: Old SIM returned to stock
    Expired --> Inventory: Return to stock
    Active --> Damaged: Physical damage
    Damaged --> Swapping: Replace with new ICCID

    State transition rules: - Inventory → Registered: Agent completes KYC and enters ICCID; emits iccid.registered - Registered → Active: Device first attach; emits iccid.activated - Active → Suspended: Payment failure or admin action; device denied attach; emits iccid.suspended - Active → Lost: User reports; block attach; emit iccid.lost - Lost/Damaged → Swapping: Initiate swap; old ICCID invalidated; emit iccid.swap_initiated - Registered → Expired: 14 days no activation; return to inventory; emit iccid.expired

    Band compatibility¶

    Required bands for Zamfara pilot: - 2G: GSM 900/1800 MHz - 3G: WCDMA Band 1 (2100 MHz) - 4G: LTE Band 3 (1800 MHz), Band 8 (900 MHz)

    Device validation: - Verify device supports at least one MNO band used by partner - Test attach on both Aku edge and MNO fallback - Document IMEI prefix patterns for auto-validation

    Known limitations¶

    • SIM lock: Devices locked to other carriers cannot attach; require unlock or use unlocked device
    • VoLTE support: Limited on feature phones; fallback to circuit-switched voice
    • USSD rendering: Some devices have character limits (160 chars); keep menus concise
    • PIN retry: Feature phones may permanently lock after 3 wrong PINs; provide PUK proactively

    Compatibility validation checklist¶

    ☐ Device supports required MNO bands (2G/3G/4G)
☐ Device is unlocked or locked to compatible MNO
☐ SIM slot accepts standard mini/micro/nano SIM
☐ Device can render USSD menus (for pSIM policy updates)
☐ IMEI is valid and not blacklisted

    Expansion roadmap¶

    • Q2 2026: Add support for more KaiOS and feature phone models
    • Q3 2026: Test ruggedized phones for field technician use
    • Q4 2026: Validate tablets with SIM slots for classroom deployments

    Feature parity with eSIM¶

    • Shared policy and flow diagram: see index.md.
    • Voice/SMS/Data policies mirror eSIM: prefer Aku edge/super-hub breakout; fallback to partner MNO.
    • Rating/billing: same prepaid/postpaid bundles and zero-rated education content.
    • Security: PIN/PUK enforced; optional IMSI lock to reduce SIM theft abuse.
    • Observability: activation logs, attach success, call/SMS CDRs, and data usage feed the same KPI dashboards.

    Operations¶

    • Replace: quick swap flow for damaged SIMs; retains subscriber MSISDN and balance.
    • Suspend/Resume: handled in IG-Hub with immediate propagation to gateway HLR/HSS tables.
    • Distribution: track stock at field offices; trigger restock alerts when low.
    • USSD/SMS: balance checks, bundle purchase, diagnostics entry point when data is unavailable.

    KPIs¶

    • Activation success rate (first attempt).
    • Attach/registration success by device class.
    • % traffic staying on Aku breakout vs partner MNO.
    • SIM swap MTTR (lost/damaged to restored service).

    GW->>MNO: No edge coverage, route via MNO

    Data session flow (feature phone constraints)¶

    sequenceDiagram
    participant DEV as Device (pSIM)
    participant GW as Gateway
    participant EDGE as Edge Hub

    GW->>EDGE: Policy: prefer local breakout

    QoS Rules¶

    Tier definitions (pSIM parity with eSIM)¶

    Tier Use Case Max Throughput Priority Throttle After
    Zero-rated Education content (allow-listed domains) 5 Mbps High Never
    Standard General browsing, social media 3 Mbps Medium 5 GB/month
    Premium HD video, large downloads 10 Mbps High 20 GB/month
    Voice Circuit/VoIP calls 128 Kbps (G.711) Highest Never

    QoS enforcement (feature phone considerations)¶

    • Zero-rated content: Bypass metering; always high priority; domain/IP allow-list maintained by policy engine.
    • Fair-use throttling: After bundle exhausted, reduce to 512 Kbps; notify via USSD/SMS.
    • Voice priority: Circuit-switched or VoLTE packets prioritized; guaranteed 128 Kbps minimum.
    • Congestion: Apply weighted fair queuing; voice > zero-rated > premium > standard.
    • 2G/EDGE devices: Pre-apply compression for WAP content; reduce QoS targets to match radio limits.

    Policy updates (limited OTA)¶

    • USSD/SMS menus for bundle changes and diagnostics.
    • Policy refresh on next attach (no real-time push for most feature phones).
    • SIM Toolkit (STK) menus where supported for balance/bundle access.
    • Fallback: cached last-known policy; update on next successful attach.

    Dashboards¶

    KPI Dashboard (Grafana): - Activation success rate (target: >97% for feature phones, >98% for smartphones) - Attach success rate by device class (target: >98%) - USSD session success rate (target: >95%) - Breakout share: Aku vs MNO (target: >65% Aku for pSIM) - Active ICCIDs gauge - Inventory available gauge (alert when <1000) - SIM swap MTTR (target: <30 min)

    Operational Dashboard: - Registrations by agent and location - Attach attempts by RAT (2G/3G/4G) and cell - ICCID state distribution (inventory, registered, active, suspended, lost) - Error rate by type (pin_locked, kyc_failed, band_incompatible) - Voice/data usage by routing (local vs MNO)

    Inventory Dashboard: - Stock levels by field office - Consumption rate (ICCIDs/day) - Restock alerts (trigger at <1000 per location) - Damaged/returned SIM tracking

    Alerting rules: - Activation success rate <95% for 15 min → Alert field ops - PIN lock rate >5% for 10 min → Alert training team (agent issue?) - Inventory <1000 at any location → Alert supply chain - SIM swap MTTR >60 min → Alert field supervisor - Attach failure rate >3% for 10 min → Alert network ops
