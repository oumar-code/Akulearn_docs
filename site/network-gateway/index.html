    Aku Network Gateway¶

    Purpose¶

    The network gateway unifies connectivity for Aku devices, providing consistent policy control for both embedded eSIM and physical SIM users, including non-smartphone/feature-phone segments.

    Components¶

    • Policy Engine: enforces preferred Aku breakout, falls back to partner MNOs when required.
    • Provisioning: handles profile creation and lifecycle for eSIM and ICCID management for physical SIM.
    • Identity: integrates with IG-Hub IdP for device attestation and session tokens.
    • Observability: exports attach/registration metrics, call/SMS CDRs, and policy conformance KPIs.

    Shared network selection policy (eSIM and physical SIM)¶

    flowchart LR
    Device[Device (eSIM or pSIM)] --> GW[Network Gateway]
    GW -->|Coverage? yes| EDGE[Aku Edge / Super Hub]
    GW -->|Coverage? no| MNO[Partner MNO]
    EDGE --> KPI[Log KPIs: attach, breakout share]
    MNO --> KPI
    GW --> POLICY[Policy updates OTA/USSD]
    POLICY --> Device
    • Default: prefer Aku Edge/Super-Hub breakout for voice/data; degrade to partner MNO when unavailable.
    • Policy delivery: OTA for eSIM; USSD/SMS menus for physical SIM where device constraints apply.
    • Parity: rating/billing bundles and zero-rated education access remain aligned across SIM types.

    Failure modes & fallbacks¶

    flowchart TD
    D[Device] --> ATT[Attach]
    ATT -->|Success| GW[Network Gateway]
    ATT -->|Fail| RETRY{Retry budget?}
    RETRY -->|Yes| BACKOFF[Backoff + alternate RAT/PLMN]
    BACKOFF --> ATT
    RETRY -->|No| SMS[USSD/SMS diagnostics]
    GW --> EDGE[Aku Edge/Super Hub]
    GW -->|No Edge Coverage| MNO[Partner MNO]
    GW -->|Policy push fails| CACHE[Use cached last-known policy]
    CACHE --> RETRY

    GW -->|SIM Swap Triggered| SWAP[Invalidate old credential]
    SWAP --> NEWCRED[Provision new eSIM/pSIM]
    NEWCRED --> ATT

    EDGE -->|Throttling Active| QOS[Apply QoS limits]
    MNO -->|Throttling Active| QOS
    QOS --> CONTINUE[Continue with limited rate]

    GW -->|Device Suspended| BLOCK[Block attach]
    BLOCK --> USSD[USSD reactivation prompt]

    Failure mode details¶

    • No Coverage: Limit attach retries; rotate PLMN/RAT list before declaring no-service.
    • Policy Fetch Failure: Keep last-known policy and retry on exponential backoff.
    • Complete Outage: When both edge and MNO are unreachable, trigger USSD/SMS diagnostic prompt where applicable.
    • SIM Swap: Invalidate old credential (profileId/ICCID), provision new target, push policy.
    • Throttling: Apply QoS limits when bundle exhausted or fair-use exceeded; notify via USSD.
    • Suspension: Block attach; surface USSD menu for balance/reactivation.

    Provisioning & lifecycle (shared view)¶

    sequenceDiagram
    participant Admin
    participant IG as IG-Hub
    participant NG as Network Gateway
    participant Dev as Device (eSIM/pSIM)
    Admin->>IG: Create subscription + policy
    IG->>NG: Push profile (eSIM) or ICCID policy (pSIM)
    Admin->>Dev: Deliver QR/activation (eSIM) or SIM card (pSIM)
    Dev->>NG: Attach + register
    NG->>Dev: Policy + breakout prefs
    NG->>IG: Report activation + KPIs
    • eSIM: profile/QR delivery; pSIM: ICCID stock + PIN/PUK handover.
    • Swaps: IG-Hub triggers new profile/ICCID mapping; gateway invalidates old credential.
    • Suspend/Resume: policy flag propagated to gateway; device denied attach when suspended.

    subgraph Device Layer
        DEV[Device]
    end

    subgraph Gateway Layer

        HLR[HLR/HSS]
    end

        EDGE[Aku Edge Hub]
        SUPER[Aku Super Hub]
    end

    subgraph Partner Layer
        MNO[Partner MNO]
    end

    DEV -->|1. Attach request| GW
    GW -->|2. Auth check| HLR
    HLR -->|3. Subscriber valid| POLICY
    POLICY -->|4. Select breakout| EDGE
    POLICY -->|4. Fallback| MNO
    EDGE -->|5. Local routing| DEV
    MNO -->|5. External routing| DEV
    POLICY -->|QoS enforcement| EDGE
    POLICY -->|QoS enforcement| MNO

    Enforcement layers¶

    1. Gateway (attach/registration): Device auth, subscriber validation, policy selection
    2. Policy Engine: Network selection (edge vs MNO), QoS tier assignment, throttling rules
    3. Edge/Super Hub: Local breakout routing, zero-rated content filtering, call/SMS termination
    4. Partner MNO: External interconnect, roaming fallback, MMS/SMS delivery
