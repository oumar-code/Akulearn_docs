    Akulearn API Specifications¶

    Purpose¶

    API specifications define the contract between Akulearn's backend services and client applications (mobile, web, projector OS). They ensure consistency, enable parallel development, and support automated testing and documentation. Akulearn uses the OpenAPI (Swagger) standard for all RESTful APIs.

    Below are example OpenAPI specifications for the User Authentication and Content APIs.

    User Authentication API (OpenAPI Example)¶

    openapi: 3.0.3
info:
  title: Akulearn User Authentication API
  version: 1.0.0
  description: API for user registration, login, and JWT authentication.
servers:
  - url: https://api.akulearn.com/v1
paths:
  /auth/register:
    post:
      summary: Register a new user
      requestBody:
        required: true
        content:
          application/json:
            schema:
              type: object
              properties:
                email:
                  type: string
                password:
                  type: string
                role:
                  type: string
      responses:
        '201':
          description: User registered successfully
        '400':
          description: Invalid input
  /auth/login:
    post:
      summary: Login and receive JWT
      requestBody:
        required: true
        content:
          application/json:
            schema:
              type: object
              properties:
                email:
                  type: string
                password:
                  type: string
      responses:
        '200':
          description: Login successful, JWT returned
        '401':
          description: Invalid credentials

    Content API (OpenAPI Example)¶

    openapi: 3.0.3
info:
  title: Akulearn Content API
  version: 1.0.0
  description: API for uploading, retrieving, and managing educational content.
servers:
  - url: https://api.akulearn.com/v1
paths:
  /content/upload:
    post:
      summary: Upload new educational content
      requestBody:
        required: true
        content:
          multipart/form-data:
            schema:
              type: object
              properties:
                file:
                  type: string
                  format: binary
                title:
                  type: string
                subject:
                  type: string
                grade:
                  type: string
      responses:
        '201':
          description: Content uploaded successfully
        '400':
          description: Invalid input
  /content/{id}:
    get:
      summary: Retrieve content by ID
      parameters:
        - name: id
          in: path
          required: true
          schema:
            type: string
      responses:
        '200':
          description: Content retrieved
        '404':
          description: Content not found

    For the full API specifications and additional endpoints, see the OpenAPI documentation in this repository.

    Network Gateway & SIM Management APIs (OpenAPI excerpts)¶

    openapi: 3.0.3
info:
  title: Network Gateway SIM Management API
  version: 1.0.0
servers:
  - url: https://api.akulearn.com/network-gateway
security:
  - bearerAuth: []
paths:
  /esim/profiles:
    get:
      summary: List eSIM profiles with pagination
      parameters:
        - in: query
          name: page
          schema: {type: integer, default: 1}
        - in: query
          name: pageSize
          schema: {type: integer, default: 20, maximum: 100}
        - in: query
          name: status
          schema: {type: string, enum: [active, suspended, deleted]}
        - in: query
          name: deviceId
          schema: {type: string}
      responses:
        '200':
          description: List of profiles
          content:
            application/json:
              schema:
                type: object
                properties:
                  data: {type: array, items: {type: object}}
                  pagination:
                    type: object
                    properties:
                      page: {type: integer}
                      pageSize: {type: integer}
                      total: {type: integer}
        '400': {description: Invalid query parameters}
        '429': {description: Rate limited}
        '500': {description: Internal server error}
    post:
      summary: Create eSIM profile with policy
      requestBody:
        required: true
        content:
          application/json:
            schema:
              type: object
              properties:
                deviceId:
                  type: string
                policyId:
                  type: string
                breakoutPreference:
                  type: string
                  enum: [aku-first, mno-first]
      responses:
        '201': {description: Profile created}
        '400': {description: Invalid input}
        '409': {description: Conflict (device already has active profile)}
        '429': {description: Rate limited}
        '500': {description: Internal server error}
        '503': {description: Service temporarily unavailable}
  /esim/profiles/{profileId}/status:
    patch:
      summary: Suspend/Resume/Delete eSIM profile
      parameters:
        - in: path
          name: profileId
          required: true
          schema: {type: string}
      requestBody:
        required: true
        content:
          application/json:
            schema:
              type: object
              properties:
                action:
                  type: string
                  enum: [suspend, resume, delete]
      responses:
        '200': {description: Status updated}
        '404': {description: Not found}
        '409': {description: Conflict (invalid state transition)}
        '429': {description: Rate limited}
        '500': {description: Internal server error}
        '503': {description: Gateway service unavailable}

  /psim/iccids:
    get:
      summary: List physical SIM ICCIDs with pagination
      parameters:
        - in: query
          name: page
          schema: {type: integer, default: 1}
        - in: query
          name: pageSize
          schema: {type: integer, default: 20, maximum: 100}
        - in: query
          name: status
          schema: {type: string, enum: [active, suspended]}
        - in: query
          name: subscriberId
          schema: {type: string}
      responses:
        '200':
          description: List of ICCIDs
          content:
            application/json:
              schema:
                type: object
                properties:
                  data: {type: array, items: {type: object}}
                  pagination:
                    type: object
                    properties:
                      page: {type: integer}
                      pageSize: {type: integer}
                      total: {type: integer}
        '400': {description: Invalid query parameters}
        '429': {description: Rate limited}
        '500': {description: Internal server error}
    post:
      summary: Register physical SIM ICCID + policy
      requestBody:
        required: true
        content:
          application/json:
            schema:
              type: object
              properties:
                iccid:
                  type: string
                subscriberId:
                  type: string
                policyId:
                  type: string
      responses:
        '201': {description: ICCID registered}
        '400': {description: Invalid input}
        '409': {description: ICCID already registered}
        '429': {description: Rate limited}
        '500': {description: Internal server error}
        '503': {description: Service temporarily unavailable}

  /subscriptions/{id}/swap:
    post:
      summary: Swap subscription between eSIM and pSIM
      parameters:
        - in: path
          name: id
          required: true
          schema: {type: string}
      requestBody:
        required: true
        content:
          application/json:
            schema:
              type: object
              properties:
                targetType:
                  type: string
                  enum: [esim, psim]
                targetId:
                  type: string
                  description: profileId for eSIM or ICCID for pSIM
      responses:
        '200': {description: Swap scheduled}
        '404': {description: Subscription not found}
        '409': {description: Swap not allowed in current state}
        '429': {description: Rate limited}
        '500': {description: Internal server error}
        '503': {description: Service temporarily unavailable}

   /policies/{id}/push:
    post:
      summary: Push latest network selection policy to device
      parameters:
        - in: path
          name: id
          required: true
          schema: {type: string}
      responses:
        '202': {description: Policy push enqueued}
        '404': {description: Not found}
        '409': {description: Push already in-flight}
        '429': {description: Rate limited}
        '500': {description: Internal server error}
        '503': {description: Gateway queue unavailable}

   /ussd/sessions:
    post:
      summary: Start USSD session for pSIM diagnostics/balance
      requestBody:
        required: true
        content:
          application/json:
            schema:
              type: object
              properties:
                msisdn:
                  type: string
                menu:
                  type: string
                  enum: [balance, bundles, diagnostics]
      responses:
        '200': {description: Session started}
        '404': {description: MSISDN not found}
        '409': {description: Session already active}
        '429': {description: Rate limited}
        '500': {description: Internal server error}
        '503': {description: USSD gateway unavailable}

  /esim/profiles/batch:
    post:
      summary: Bulk create eSIM profiles
      requestBody:
        required: true
        content:
          application/json:
            schema:
              type: object
              properties:
                profiles:
                  type: array
                  maxItems: 100
                  items:
                    type: object
                    properties:
                      deviceId: {type: string}
                      policyId: {type: string}

      responses:

            application/json:
              schema:
                type: object
                properties:
                  batchId: {type: string}
                  status: {type: string, enum: [queued, processing]}
                  totalRequests: {type: integer}
        '400': {description: Invalid input or batch size exceeded}
        '429': {description: Rate limited}
        '500': {description: Internal server error}

  /esim/profiles/batch/{batchId}:
    get:
      summary: Get batch operation status
      parameters:
        - in: path
          name: batchId
          required: true
          schema: {type: string}
      responses:
        '200':
          description: Batch status retrieved
          content:
            application/json:
              schema:
                type: object

  /psim/iccids/batch:
    post:
      summary: Bulk register physical SIM ICCIDs
      requestBody:
        required: true

          application/json:
            schema:
              type: object
              properties:
                iccids:
                  type: array

                      policyId: {type: string}

    Error codes (draft)¶

    • invalid_input: schema validation failed.
    • policy_not_found: referenced policyId is missing.
    • device_not_found: deviceId not known to IG-Hub.
    • iccid_conflict: ICCID already bound to another subscriber.
    • swap_not_allowed: subscription state prevents swap (e.g., suspended).
    • policy_push_failed: gateway could not enqueue policy update.
    • internal_error: unexpected server-side failure; includes requestId for tracing.
    • service_unavailable: gateway/HLR/HSS temporarily down; includes retryAfter hint.
    • database_error: persistent storage unavailable.
    • gateway_timeout: upstream gateway did not respond in time.
